# Case Study: Production DB Performance Engineering & Engine Modernization

## Context

Optimizer Team 소속으로 엔터프라이즈 고객사의 **운영 데이터베이스 이슈 대응**을 담당하였다.  
평균적으로 주 2건 이상의 고객사 운영 이슈를 처리하였으며, 대부분 **성능 관련 문제**였다.

이 과정에서 실행 계획(Execution Plan), 통계(Statistics), 그리고 DB 엔진 내부 구조에 대한 깊은 이해가 요구되었다.

또한 동시에 **기존 C 기반으로 작성된 레거시 엔진 모듈을 C++로 현대화하는 작업**에도 참여하였다.

---

## Problem

### 1️⃣ 고객사 운영 환경 성능 문제

- 운영 환경에서 쿼리 성능 저하 발생
- 통계 정보, 스키마 구조, 아키텍처 한계로 인해 비효율적인 실행 계획 생성
- 일부 문제는 단순 SQL 튜닝만으로 해결 불가능

### 2️⃣ 레거시 엔진 코드 유지보수 문제

- 핵심 엔진 모듈이 레거시 C 코드로 작성
- 모듈화 수준이 낮아 유지보수 어려움
- Optimizer 로직 확장 및 개선 작업이 점점 어려워짐

---

## Implementation

### 🔹 고객사 DB 성능 문제 해결

#### 1. Query Tuning

- 실행 계획 및 Cardinality 추정 분석
- 통계 정보 및 인덱스 전략 조정
- 필요 시 복잡한 쿼리 구조 재작성

#### 2. Database Architecture 개선

- 스키마 구조 재설계 제안
- 인덱스 전략 재설계
- 파티셔닝 전략 최적화
- 워크로드 분산 구조 개선

#### 3. Engine-Level Patch 개발

- SQL 튜닝만으로 해결되지 않는 문제에 대해 엔진 수준 수정 수행
- Optimizer 로직 또는 통계 처리 방식 수정
- 패치된 DB 바이너리를 고객사 환경에 제공하여 근본 원인 해결

---

### 🔹 레거시 엔진 현대화 (C → C++)

- 기존 C 기반 모듈을 C++ 구조로 리팩토링
- 모듈 구조 개선 및 코드 유지보수성 향상
- 객체지향 설계 원칙 적용
- 메모리 안정성 및 코드 가독성 개선
- 향후 Optimizer 기능 확장을 위한 구조 기반 마련

---

## Outcome

- 여러 엔터프라이즈 고객사의 **고위험 운영 성능 문제 해결**
- 주요 케이스에서 **쿼리 실행 시간 단축**
- 실제 워크로드 환경에서 **Optimizer 안정성 향상**
- 핵심 엔진 모듈의 **유지보수성과 확장성 개선**
- DBA 수준의 운영 문제 해결과 **DB 엔진 개발 역량을 동시에 입증**

결과적으로 엔터프라이즈 RDBMS 환경에서 **시스템 안정성과 고객 만족도 향상**에 기여하였다.
